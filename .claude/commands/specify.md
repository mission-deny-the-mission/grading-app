Define feature requirements and user stories in a structured specification.

This command creates detailed specifications for new features, including user stories, acceptance criteria, and success metrics.

Usage:
/specify <feature description>

Examples:
/specify Add real-time notifications to the grading dashboard so users can see progress updates as documents are being processed
/specify Implement bulk export functionality for grading results in CSV and PDF formats
/specify Create a user role management system with different permission levels
